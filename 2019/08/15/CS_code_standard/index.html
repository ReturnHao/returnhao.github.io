命名规则

术语定义

Pascal命名方法：将标识符的首字母和后面连接的每个单词的首字母都大写。例如：PascalRule，简记为Pascal法。

Camel命名方法：标识符首字母小写，而后面每个连接的单词都大写。例如：camelRule，简记为Camel法。

命名规则

文件命名：遵循Pascal法，扩展名为.cs。

类、方法、枚举、属性、事件、接口：遵循Pascal法。

变量、参数：遵循Camel法。

常量：所有字母都是大写，单词间用下划线连接。例如：public const int AGE_OF_PERSON = 20;

命名尽量使用简明扼要的英文关键词表述，布尔变量需使用is开头，这样可以明确含义。对于函数局部循环变量可以使用i，j，k，如果有明确含义的变量应尽量使用命名表述清楚。

注释方案

注释概述

目的：注释的目的是为了提高代码可读性，因此无意义或容易产生歧义的注释是必须坚决避免的。

原则：初次编写代码时就要及时写好注释（因为你很有可能后面不会再有时间去注释，或者过一段时间会忘记）；每次代码的修改后，都必须更新注释，确保注释准确；注释请严格按照统一的风格和标准，保持代码高可读性。
