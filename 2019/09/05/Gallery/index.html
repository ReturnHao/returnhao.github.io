Aim

本项目旨在实现一个基于SLAM和AR的画廊观赏项目。UI方面要求：

  • 实现场景变化功能
  • 场景变化通过一个随机位置的传送门实现
  • 每次场景变化都需要保证画廊生成在用户当前面对位置的正前方
  • 用户在传送门附近会出现跟随用户的引导箭头，远离时消失

Requirement

  • Unity 2017.4.20f2

Project Process

本项目的过程分为三部分，第一部分为场景切换，由Scene Generator.cs控制，第二部分为用户交互，同样由Scene Generator.cs控制，最后一部分为画廊生成，由Galley.cs控制。

Controller

人机交互部分，我们需要考虑以下两方面，首先是动态箭头的显示，其次是进入下一个场景的方法。

Dynamic Arrow

这里我们用 Dynamic Arrow 来保存整个 Arrow 组件，其包含了用于显示文字的Text Wrapper和用于生成箭头的 arrowPrefab 。控制箭头生成的函数为 AdjustArrow()

箭头生成的方法和传送门的生成方式大体一致，我们将Arrow生成在距离眼睛平面垂直距离一米的平面上。

我们用 LateUpdate() 控制箭头的出现（以保证其不频闪），同时通过判断用户和传送门的距离即角度函数*IsPortalAppear()*来决定箭头是否被激活。

Enter Controller

进入下一个场景需要判定两个满足条件，一个是用户看到了箭头，即用户出现在传送门附近，另一个是，用户是否在传送门旁。
判断用户和传送门的关系的核心是两者forward方向以及，两者的距离：

如果两者所成角度在[-30,30]，同时距离不超过6m的话，我们就默认用户在传送门边上了。

当用户在门边的时候，可以触发进入下一个场景的动作，这个动作需要用户长按确认键。我们通过检测每一次按下的时间和当前时间的差来判定。

至此，SceneController部分结束。

Gallery

Gallery部分主要由两部分组成，一部分是初始模式的画的生成（这部分不完全由我负责，在此就略过不提了），另一部分是*Spawn()*函数，用以在特定的位置生成特定的画廊。

至此，Gallery的生成解析完毕。
