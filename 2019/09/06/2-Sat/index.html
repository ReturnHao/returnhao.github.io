2-Satisfiability

Posted on 2019-09-06 | Post modified: 2019-09-10 | In Algorithm

2-SAT is a special case of Boolean Satisfiability Problem and can be solved
in polynomial time.

Boolean Satisfiability Problem

Boolean Satisfiability or simply SAT is the problem of determining if a Boolean formula is satisfiable or unsatisfiable.

Satisfiable :

If the Boolean variables can be assigned values such that the formula turns out to be TRUE, then we say that the formula is satisfiable.

Unsatisfiable :

If it is not possible to assign such values, then we say that the formula is unsatisfiable.

Examples :

  • F=A∧B¯F = A\land\bar{B}F=A∧​B​¯​​, is satisfiable, because A = TRUE and B = FALSE makes F = TRUE.

  • G=A∧A¯G = A\land\bar{A}G=A∧​A​¯​​, is unsatisfiable, because:

    A A¯\bar{A}​A​¯​​ G
    TRUE FALSE FALSE
    FALSE TRUE FALSE

To understand this better, first let us see what is Conjunctive Normal Form (CNF) or also known as Product of Sums (POS).

CNF : CNF is a conjunction (AND) of clauses, where every clause is a disjunction (OR).

Now, 2-SAT limits the problem of SAT to only those Boolean formula which are expressed as a CNF with every clause having only 2 terms (also called 2-CNF ).

Thus, Problem of 2-Satisfiabilty can be stated as:

Given CNF with each clause having only 2 terms, is it possible to assign such values to the variables so that the CNF is TRUE?

Approach for 2-SAT Problem

For the CNF value to come TRUE, value of every clause should be TRUE. Let one of the clause be (A∨B)(A\lor B)(A∨B).

(A∨B)(A\lor B)(A∨B) = TRUE

  • if A = FALSE, B must be TRUE i.e. (A¯⇒B)(\bar{A}\Rightarrow B)(​A​¯​​⇒B)
  • if B = FALSE, A must be TRUE i.e. (B¯⇒A)(\bar{B}\Rightarrow A)(​B​¯​​⇒A)

Thus,

(A∨B)(A\lor B)(A∨B) = TRUE is equivalent to (A¯⇒B)∧(B¯⇒A)(\bar{A}\Rightarrow B)\land (\bar{B}\Rightarrow A)(​A​¯​​⇒B)∧(​B​¯​​⇒A)

Now, we can express the CNF as an Implication. So, we create an Implication Graph which has 2 edges for every clause of the CNF.

(A∨B)(A\lor B)(A∨B) is expressed in Implication Graph as edge(A¯⇒B)edge(B¯⇒A)edge(\bar{A}\Rightarrow B) edge(\bar{B}\Rightarrow A)edge(​A​¯​​⇒B)edge(​B​¯​​⇒A)

Thus, for a Boolean formula with ‘m’ clauses, we make an Implication Graph with:

  • 2 edges for every clause i.e. ‘2m’ edges.
  • 1 node for every Boolean variable involved in the Boolean formula.

Note: The implication (if A then B) is equivalent to its contrapositive (if B¯\bar{B}​B​¯​​ then A¯\bar{A}​A​¯​​).

Now, consider the following cases:

  • CASE 1: If edge(X⇒X¯)edge(X \Rightarrow \bar{X})edge(X⇒​X​¯​​) exists in the graph
    This means (X⇒X¯)(X \Rightarrow \bar{X})(X⇒​X​¯​​)
    If X = TRUE, X¯\bar{X}​X​¯​​ = TRUE, which is a contradiction.
    But if X = FALSE, there are no implication constraints.
    Thus, X = FALSE i.e. X¯\bar{X}​X​¯​​ = TRUE

  • CASE 2: If edge(X¯⇒X)edge(\bar{X} \Rightarrow X)edge(​X​¯​​⇒X) exists in the graph
    This means (X¯⇒X)(\bar{X} \Rightarrow X)(​X​¯​​⇒X)
    If X¯\bar{X}​X​¯​​ = TRUE, X = TRUE, which is a contradiction.
    But if X¯\bar{X}​X​¯​​ = FALSE, there are no implication constraints.
    Thus, X¯\bar{X}​X​¯​​ = FALSE i.e. X = TRUE

  • If edge(X⇒X¯)&edge(X¯⇒X)edge(X \Rightarrow \bar{X}) \& edge(\bar{X} \Rightarrow X)edge(X⇒​X​¯​​)&edge(​X​¯​​⇒X) both exist in the graph
    One edge requires X to be TRUE and the other one requires X to be FALSE.
    Thus, there is no possible assignment in such a case.

CONCLUSION:

If any two variables X and X¯\bar{X}​X​¯​​ are on a cycle i.e. path (A¯⇒B)&path(B¯⇒A)(\bar{A} \Rightarrow B) \& path(\bar{B} \Rightarrow A)(​A​¯​​⇒B)&path(​B​¯​​⇒A) both exists, then the CNF is unsatisfiable. Otherwise, there is a possible assignment and the CNF is satisfiable.
Note here that, we use path due to the following property of implication:
If we have (A⇒B)&(B⇒C)(A \Rightarrow B) \& (B \Rightarrow C)(A⇒B)&(B⇒C), then A⇒CA \Rightarrow CA⇒C
Thus, if we have a path in the Implication Graph, that is pretty much same as having a direct edge.

CONCLUSION FROM IMPLEMENTATION POINT OF VIEW:

If both X and X¯\bar{X}​X​¯​​ lie in the same SCC (Strongly Connected Component), the CNF is unsatisfiable.
A Strongly Connected Component of a directed graph has nodes such that every node can be reach from every another node in that SCC.
Now, if X and X¯\bar{X}​X​¯​​ lie on the same SCC, we will definitely have path(A¯⇒B)&path(B¯⇒A)path(\bar{A} \Rightarrow B) \& path(\bar{B} \Rightarrow A)path(​A​¯​​⇒B)&path(​B​¯​​⇒A) present and hence the conclusion.

Checking of the SCC can be done in O(E+V) using the Kosaraju’s Algorithm or Tarjan’s Algorithm

Common construction notes

Beside the example (A∨B)(A\lor B)(A∨B) we mentioned, there still exist some more expressions.

Mostly they can be implemented by simple logical based expression easily. For example,

  • (A∧B)(A\land B)(A∧B)=TRUE means both A and B shall be TRUE i.e. edge(A¯⇒A)&edge(B¯⇒B)edge(\bar{A}\Rightarrow A) \& edge(\bar{B}\Rightarrow B)edge(​A​¯​​⇒A)&edge(​B​¯​​⇒B)
  • ¬(A∧B)\neg(A\land B)¬(A∧B)=TRUE means at least one of A and B shall be FALSE i.e. edge(B⇒A¯)&edge(A⇒B¯)edge(B\Rightarrow \bar{A}) \& edge(A\Rightarrow \bar{B})edge(B⇒​A​¯​​)&edge(A⇒​B​¯​​)

Generally, you can see this graph is symmetric, and this is the basis of its correctness proof.

Solution

Sometimes we need to output one possible solution(Notice there may be exist not only one solution).

Here we use the conception of Inverse Graph, which inversed all directed edges in original graph GGG to a new graph GiG_iG​i​​.

Since we have eliminate all possible circle in G by SCC algorithm, we can determine Topological Order of this new directed acyclic graph G’. Topological Order reflect the affect factor of all the points. Those points with smaller order fact more points than those larger order points. Specifically, consider X→X¯X\rightarrow \bar{X}X→​X​¯​​, which means if we choose X, we shall also choose X¯\bar{X}​X​¯​​. However, from our conclusion, this edge means we shall never choose X. Therefore, those who with larger order(X¯\bar{X}​X​¯​​) shall be consider first.

Back to our SCC Algorithm, Kosaraju & Tarjan, both generate topological order during the process with different generate rules — Tarjan’s SCC numbers equal to Inverse Graph’s topological order and Kosaraju, generate SCC numbers as Origin Graph.

Hence, for Tarjan, you should choose those with smaller SCC order numbers and for Kosaraju, you should choose those with larger SCC order numbers.
